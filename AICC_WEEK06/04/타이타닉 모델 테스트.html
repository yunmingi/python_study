1. 타이타닉 생존자 분류 모델 테스트¶

(1) 목적 : 어느 모델이 타이타닉 생존자 분류에 더 적합한지 테스트하기 위함¶

(2) 테스트 모델 : 로지스틱 회귀 vs 결정 트리¶

(3) 평가 지표 : 혼동 행렬, F1-score, ROC/AUC¶

(4) 리뷰:¶

1. 데이터 전처리를 하기 전에 모델에 포함시킬 특성을 잘 선택해야¶

  • 가령 결측치가 가장 많은 deck의 경우, 상관관계만으로 봤을 때는 모델링에 있어 꽤 중요한 특성처럼 보이지만, 그 자체가 class 등과 높은 상관관계를 가지고 있어 사실상 제외해도 큰 문제는 없었다.
  • age 역시 who, adult_male 등으로 충분히 대체할 수 있으므로 굳이 결측치를 고려할 필요가 없다는 판단을 내릴 수 있었다.
  • 위처럼 유사성을 가진 특성들, class와 pclass처럼 완전히 동일한 특성들을 우선적으로 확인하고 전처리를 시작하는 편이 좋다는 것을 느낀다. #### 2. 상식과 지식을 토대로, 표면적 점수에 너무 얽매여서는 안 돼
  • 아무리 다양한 조건의 영향을 받았다 해도, 사람의 생존여부를 조건에 따라 90% 이상 맞춘다는 것은 오히려 비현실적이다. 80% 이상 예측할 수 있다는 것만으로도 충분히 안타까운 사고임을 고려한다면 점수를 높이는 데 너무 얽매일 필요가 없다는 판단을 내릴 수 있었다.

[4-1] 로지스틱 회귀 모델링¶

In [6]:
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

X = tit[['adult_male','pclass','sex_no','who_no','fare']]
y = tit['survived']

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

ss = StandardScaler()
X_train_scaled = ss.fit_transform(X_train)
X_test_scaled = ss.fit_transform(X_test)

lr = LogisticRegression(C=0.5)
lr.fit(X_train_scaled,y_train)

print(lr.score(X_train_scaled,y_train))
print(lr.score(X_test_scaled,y_test))

0.8025682182985554
0.8022388059701493
  • 규제를 0.5로 맞추었을 때 가장 적절한 점수가 나온다.
  • 컴퓨터놈이 로지스틱 회귀로 감히 사람 생사를 판단하는데 80점이나 나온다는 건 잔인한 사실이다. 더 높은 점수를 바라지 말자.

from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.tree import DecisionTreeClassifier

X = tit[['adult_male','pclass','sex_no','who_no','fare']]
y = tit['survived']

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

ss = StandardScaler()
X_train_scaled = ss.fit_transform(X_train)
X_test_scaled = ss.fit_transform(X_test)

dt = DecisionTreeClassifier(max_depth=5)
dt.fit(X_train_scaled,y_train)

print(dt.score(X_train_scaled,y_train))
print(dt.score(X_test_scaled,y_test))

0.8571428571428571
0.8208955223880597
  • max_depth=5를 해주니 적절한 점수가 나온다.
  • 예상은 했지만 결정 트리는 더 잔인한 놈이다.
